1. Clone

Permite clonar un repositorio remoto a un workspace.

git clone [path]

Ejemplo:

git clone https://github.com/jhonnyesquivel/git-basics.git

2. Checkout

Permite crear/cambiar la rama en la que nos encontramos

git checkout [-b] <branch>

El parametro opcional -b le indica al terminal que en caso de que el branch no exsita localmente lo cree.

Si no se agrega -b: Si el branch se encuentra localmente, simplemente hara un cambio de rama. Si el branch no existe, mostrará un error.

Ejemplo:

git checkout -b feature/add
git checkout feature/add

4. Commit

Registra todos los cambios pendientes el en repositorio local y les asigna un commit-id

git commit -m "mensaje descriptivo de los cambios"

Ejemplo:

git commit -m "se agrega el archivo index al repositorio"

8. Rebase

Aplica los commits de un branch en en el top de un branch especificádo. Requerirá ejecutar el comando: git push --force

Ejemplo:

9. Reset

Permite retrocerder hasta un commit-id específico, requerirá ejecutar el comando git push --force

git reset --hard <commit-id>

Ejemplo:

git reset --hard 3c3ef8c55539e1eb2f0a7d606f7afca3945dfcfd
